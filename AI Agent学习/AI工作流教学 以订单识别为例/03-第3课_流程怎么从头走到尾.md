# 第3课：流程怎么从头走到尾

## 这一课要讲明白的事

现在开始把口头描述变成流程图。

## 主流程

客户下单 -> 订单确认 -> 库存检查 -> 采购申请 -> 供应商询价 -> 审批 -> 下单采购 -> 到货验收 -> 付款 -> 归档

## 讲课顺序

1. 先把主流程完整读一遍
2. 再给每一步补上输入和输出
3. 再标记状态变化
4. 最后检查是否漏环节

## 讲给小白听的话

> 流程图不是为了好看，是为了让每一步都能回答三个问题：  
> 这一步从哪来、这一步要做什么、做完以后去哪。

## 课堂动作

1. 按顺序写出节点
2. 给每个节点写开始条件
3. 给每个节点写结束条件
4. 给每个节点写状态名

## 板书建议

- 输入
- 动作
- 输出
- 状态

## 小练习

- 为每一步补上开始条件和结束条件

## 本课产出

- 一张完整流程图

## 常见卡点

- 节点顺序乱
- 没写输入输出
- 状态名太抽象，别人看不懂
