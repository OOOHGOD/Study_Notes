# 【论文学习】PaddleOCR-VL-1.6：Expanding the Frontier of Document Parsing with Under-Optimized Region Refinement and Progressive Post-Training

## 📄 智能文档解析（面向大模型）
> _为大模型时代将杂乱的文档视觉信息转化为结构化数据。_

- **SOTA 级文档视觉语言模型 (VLM，Vision-Language Model，视觉语言大模型）**: 轻量级文档解析视觉语言模型 **PaddleOCR-VL-1.6 (0.9B)**。在古籍、生僻字、印章、图表等多场景能力显著，支持以 **Markdown** 和 **JSON** 格式输出结构化结果。
- **版面结构分析**：由 **PP-StructureV3** 驱动，无缝将复杂的 PDF文档 和图像转换为 **Markdown** 或 **JSON** 格式。与 PaddleOCR-VL 系列模型不同，它提供更细粒度的坐标信息，包括表格单元格坐标、文本坐标等。
- **生产级高效能**：以极小的模型体积实现商业级别的准确率。

### 文档解析为啥突然这么重要？
以前 OCR 像是「把图里的字抠出来」。现在不一样了：

- RAG 要把 PDF/扫描件变成 Markdown、JSON，才能切块、入库、检索
- 不光要文字，还要 版式、阅读顺序、公式、表格、图表、印章
- 解析错了，后面 LLM 再强也是 垃圾进、垃圾出

<details>
<summary><b>📑 什么是欠优化区域（UOR）</b>（点击展开）</summary>

这是本篇论文的核心概念。论文提出：**PaddleOCR-VL-1.5 本身已经很强，剩余错误并不是平均分布在所有样本上，而是集中在部分特殊场景中。**
这些容易出问题的区域，就叫做 **欠优化区域（Under-Optimized Regions, UOR）**。

> 通俗理解：模型不是“整体没学会”，而是“某些角落没学稳、没学够，或者学错了”。

论文将欠优化区域分为三类，可点击下方分项查看详细说明。

<details>
<summary><b>1. 边界脆弱区域（Boundary-Fragile Regions）</b></summary>

### 核心含义

输入图像只发生极小变化、语义本身没有改变，但模型输出会出现大幅跳变。

### 典型例子

同一张文档图片：

- 原图：识别完全正确
- 稍微平移 2 像素：表格结构识别错误
- 做 JPEG 压缩：公式内容遗漏
- 加一点轻微模糊：阅读顺序错乱

人眼看起来内容完全一致，但模型表现不稳定，说明该样本处于决策边界附近，一点点扰动就可能掉进错误区间。

### 检测方法

从两个视角衡量样本稳定性：

1. 训练末期多个 checkpoint 的输出是否一致
2. 同一张图片经过轻微扰动后的输出是否一致

具体流程：

- 取训练最后 8% 阶段中的 8 个 checkpoint
- 对每个样本施加 16 种轻微扰动，例如平移、压缩、噪声、模糊、缩放等
- 单个样本共得到 `8 × 16 = 128` 个输出结果
- 两两计算输出之间的 **归一化编辑距离（normalized edit distance）**，差异越大代表样本越脆弱
- 取分数最高的 top 1% 样本，划定为边界脆弱区域

### 本质

模型输出不稳定，也就是 **学得不稳**。

</details>

<details>
<summary><b>2. 覆盖稀疏区域（Coverage-Sparse Regions）</b></summary>

### 核心含义

部分样本类型在训练数据里占比极低，模型见过的同类样本太少，泛化能力不足。

### 典型场景

古籍、冷门字体、工业复杂表格、特殊版式、异形票据、小语种、密集公式页等。
这类问题不是单张图片难度高，而是整个类别的数据覆盖量不足。

### 检测方法

论文没有使用传统 K-Means 聚类，而是通过构建 **相似度图** 挖掘长尾簇：

1. 对每个训练样本提取文档特征向量
2. 计算样本之间的特征相似度，相似度高的样本之间连边，形成图结构
3. 逐步提高相似度阈值，大簇会逐步拆分，孤立的小簇会逐渐暴露
4. 筛选出规模极小、连接孤立的簇，即为覆盖稀疏区域

### 为什么不用 K-Means？

K-Means 会强制把所有样本塞进某个大类，稀有样本很容易被大簇“吞并”。
而基于图的方法通过 **局部密度** 和 **连接关系** 判断，更容易发现真实的长尾场景。

### 本质

模型见过的相关样本太少，也就是 **见得不够**。

</details>

<details>
<summary><b>3. 监督不可靠区域（Unreliable-Supervision Regions）</b></summary>

### 核心含义

训练数据的标注本身存在错误，模型基于错误标签学习，最终可能稳定地输出错误结果，而且置信度还很高。
这类问题在 OCR / 文档解析任务中很常见。比如标注的表格缺了一列、文本内容错漏，模型会被错误标签带偏。它不是学不会，而是学了错误答案。

### 修正方法

论文引入 3 个外部专家模型进行投票校验：

- Qianfan-OCR
- GLM-OCR
- MinerU2.5-Pro

校验规则非常保守：

- 若至少 1 个专家模型结果与原标签一致：保留原标签
- 若原标签与所有专家都不一致，但至少 2 个专家结果彼此一致：用 2 个专家的一致结果替换原标签
- 若所有结果均无法达成一致：标记为未解决，进入后续精细化自动标注流程

### 本质

模型学习了错误答案，也就是 **标签不准**。

</details>

<details>
<summary><b>📝 三类 UOR 总结</b></summary>

可以用三个词快速记忆：

| UOR 类型 | 中文理解 | 一句话记忆 |
|:---|:---|:---|
| Boundary-Fragile | 边界脆弱区域 | **不稳**：学得不稳 |
| Coverage-Sparse | 覆盖稀疏区域 | **不够**：见得不够 |
| Unreliable-Supervision | 监督不可靠区域 | **不准**：标签不准 |

后续论文中所有的数据引擎优化和训练策略调整，都是围绕这三类问题展开的。

</details>

<details>
<summary><b>💡 为什么这一步很重要</b></summary>

传统优化思路是线性的：

```text
模型效果不好 → 加更多数据 → 继续训练
```
这属于“广撒网”式的粗放提升。
这篇论文的核心思路是 **先诊断、后优化**：先精准定位模型薄弱点，再针对性补全短板。

类比学习备考：

- 普通方法：数学考不好 → 刷完一整本题库
- 论文方法：先整理错题本，区分是「概念不牢」「题型没见过」还是「答案本身有误」，再逐个突破

这种 **诊断式优化** 是论文最核心的思想。

</details>

## ⚛️ 实现原理

PaddleOCR-VL 整体由**版面分析**与 **VLM 识别**两个核心阶段组成。下图展示了一个简化的流程：

![](https://raw.githubusercontent.com/cuicheng01/PaddleX_doc_images/5f2c42665e6a97d5726fe553241acd79361159a0/images/paddleocr_vl_1_5/process_step_ZH.png)

### 第一阶段：版面分析

模型以整图作为输入，检测并定位图像中的各类版面元素（例如表格、公式等），同时确定其阅读顺序，并根据检测结果裁剪出对应的元素子图；

**它到底在做什么？**

PaddleOCR-VL-1.6 解决的是“文档解析”，普通 OCR 只是把图片里的字识别出来，比如：

```text
`This is a table...`
```

但文档解析要复杂得多，它要从 PDF、截图、扫描件里恢复结构：

```markdown
# 标题

正文段落...

|  表头1  |  表头2  |
|--------|---------|
| 内容 | 内容 | 内容 |
|  公式：E = mc^2   |
```

也就是说，它不仅要识别文字，还要理解：文本在哪里、阅读顺序是什么、表格结构是什么、公式是什么、图表是什么、印章是什么、标题层级是什么。所以这不是单纯 OCR，而是“**视觉 + 语言 + 结构化输出**”的任务。

**整体系统长什么样？**

论文里的 PaddleOCR-VL-1.6 有两个主要使用场景：

1. 文档解析
2. 文本定位与识别，也就是 text spotting

我们重点看文档解析，因为这是论文主线，它的实际流程大概是：

```
输入文档图片 / PDF 页面
        ↓
PP-DocLayoutV3 做版面分析
        ↓
找到不同区域：正文、表格、公式、图表、印章等
        ↓
PaddleOCR-VL-1.6-0.9B 识别每个区域
        ↓
后处理模块整理成 Markdown / JSON
```

你可以把它想象成一个流水线：先切区域，再识别区域，最后整理结构

其中：

`PP-DocLayoutV3` 负责“看页面哪里有什么”。

`PaddleOCR-VL-1.6-0.9B` 负责“理解这些区域内容”。

后处理模块负责“把结果拼成可用格式”，比如 Markdown、JSON，还包括跨页表格合并、标题层级修正等。

### 第二阶段：VLM 识别

将每个子图独立输入 VLM，生成对应的识别结果（例如 Markdown 文本），随后再按照版面分析阶段给出的顺序对各元素结果进行合并，得到整幅图像的完整解析结果。

> [!note]
> **若需使用 PaddleOCR-VL 的完整能力，必须采用版面分析与 VLM 识别协同的完整流程，而不能仅单独使用 VLM。** 后文会多次涉及相关概念，请注意区分完整的 PaddleOCR-VL 流程与其中的 VLM 组件。以 PaddleOCR-VL v1 为例，版面分析模型为 PP-DocLayoutV2，VLM 为 PaddleOCR-VL-0.9B。需要特别说明的是，“PaddleOCR-VL-0.9B” 并不是 PaddleOCR-VL 的一个模型变种，而是 PaddleOCR-VL v1 完整流程中的 VLM 组件；这与常见 LLM / VLM 的命名习惯不同，例如 Qwen2-72B 通常表示 Qwen2 系列下的一个具体模型变体。
